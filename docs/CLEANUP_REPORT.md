# Quantara AI Stabilization Cleanup Report

Date: 2026-06-01

## Audit findings

- `app.py` is the largest UI module and currently owns too many workflows. It remains active; splitting it is deferred to avoid a risky rewrite.
- `engines/data_service.py` is the central high-coupling module for universe loading, ticker resolution, caching, and provider access. It remains active.
- Basket and long-term screens were prescreening with one cached history request per symbol in the Streamlit UI, then downloading history again in the engine. This is a primary freeze and API-pressure path.
- `config.py` defaulted basket workers to `20`, which is too aggressive for yfinance and an 8GB laptop.
- `portfolio_analyzer_UPDATED.py` was the active portfolio module while tests still imported `portfolio_analyzer`. It must be consolidated under the canonical module name.
- The repository tracks Python bytecode under `__pycache__/`. These files are generated artifacts and should be removed from source control.
- Root `streamlit.out.log` and `streamlit.err.log` are stale zero-byte runtime artifacts and should be removed.
- Runtime logs under `logs/` are useful operational evidence but should not be committed. Existing files are retained locally and ignored.
- `.quantara/universe_cache.json` is a large runtime cache (about 5.4 MB). It is retained for launch speed but should not grow without bounds.

## Safe removal list

- All `__pycache__/` directories and `*.pyc` files: generated by Python and recreated automatically.
- `streamlit.out.log` and `streamlit.err.log`: zero-byte stale runtime files.

## Archive list

The following root modules have no import references from the active application. They are moved to `archive/legacy_root_modules/` rather than deleted:

- `advanced_metrics.py`
- `auto_dashboard.py`
- `chart_builder.py`
- `data_profiler.py`
- `llm_query_engine.py`
- `ollama_client.py`
- `price_predictor.py`
- `smart_dashboard.py`
- `stats_summary.py`
- `technical_indicators.py`

## Deferred cleanup

- Empty target-structure directories are retained as placeholders.
- Runtime cache pruning is not automated in this pass because cache retention affects provider load and needs a measured policy.
- `app.py` and `engines/data_service.py` should be split only after the stabilization fixes are validated.
